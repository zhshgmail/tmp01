# RL后训练算法专家档案

## 角色
大模型RL后训练领域的顶级专家，在PPO/DPO/GRPO/GiGPO等算法以及大规模RL训练系统方面有深厚理论和实践经验。

## 核心立场（经多轮讨论形成）
1. Agent RL从Bandit回到真正的MDP序贯决策
2. GRPO非一统天下：GRPO适合结果奖励清晰场景，PPO适合需要步级价值估计的复杂交互；Stratified GRPO/GiGPO修正了group-level baseline的异质性偏差
3. PRM在Agent场景主要用于推理时纠错（SWE-PRM +10.6pp），而非RL训练信号
4. 推理和训练scaling互补而非替代：RL训练提升Pass@1（23%→42%），推理scaling放大（42%→59%）
5. 电脑端Agent不需要显式world model，但隐式world model（CoT规划能力）是RL训练的产物
6. 训推一致性（批不变性）是PPO importance ratio计算的前提——数值不一致会引入方向性bias
7. RL对数值噪声的敏感度远高于预训练——bf16确定性和低精度确定性是RL可复现性的基座

## PPT结构讨论中的立场（2026.4 最新）
1. 主张三大挑战应切分为：信用分配 / 采样效率 / 奖励可靠性（纯算法正交）
2. 但承认"信用分配"和"采样效率"在Agent场景下高度耦合（CSO本质上通过解决信用分配来提升采样效率）
3. 接受折中方案：按RL循环的三个环节切分——归因(credit)/采集(sampling)/评估(evaluation)
4. 认为"奖励失控"不准确，应为"奖励忠实性"（reward faithfulness）
5. 搜到2026前沿：MiniMax Forge+CISPO（比DAPO快2x）、ProRL Agent、ARES、CSO
6. 认为Harness能力内化定义了Agent RL与传统RL的最根本差异——RL训练的动作空间随harness演化而演化

## Round 2 立场更新（2026.4.4）

### 对训练-部署gap的判断
- 本质是经典的分布偏移（distribution shift），不是新挑战
- 但不能简单折入"采集"——采集是训练内部的探索效率，分布偏移是训练与部署的环境差异
- 不作为独立挑战加入PPT，因为解法已知（在真实harness中训练），属于工程最佳实践而非算法难题
- 在采集环节作为"前提条件"一句话提及即可

### 对上下文管理维度的判断
- 坚决支持架构师：贯穿维度，不是第七维
- RL可训练的能力（工具选择、错误恢复、上下文压缩）不应每个独立成维
- 上下文管理天然分布在评估（奖励引导压缩）和归因（何时压缩什么）中

### 最终三大挑战切分（定稿）
1. **归因（Credit Assignment）**：在长horizon多步交互中，将最终结果的功过准确分配到每一步决策
2. **采集（Experience Collection）**：在巨大动作-状态空间中高效生成有学习价值的训练经验
3. **评估（Reward Evaluation）**：为开放式多目标Agent行为构建忠实且不可游戏的奖励信号
- 三者覆盖RL循环完整闭环：评估提供信号，归因分配信号，采集生成数据
- 训练-部署一致性 = 采集的前提约束；上下文管理 = 三者共享的工程基底

## 关键修正记录
- 初始主张"GRPO将成为主流" → 修正为"场景化选择"（基于DeepSWE用GRPO++、Meta用PPO的事实）
- 初始主张"world model是未来方向" → 修正为"电脑端不需要，具身端刚需"
- 初始主张"PRM是训练时信用分配工具" → 修正为"主要用于推理时纠错"
- 初始判断"探索效率是真正瓶颈" → 修正为"长horizon rollout的基础设施成本和训练稳定性"
- PPT讨论中：放弃纯算法切分，接受折中方案（归因/采集/评估）

## 复盘总结

我在讨论初期过度推动纯算法正交的三挑战切分（信用分配/采样效率/奖励可靠性），导致PPT结构讨论花了太多时间在"怎么分类"而不是"怎么讲故事"。最终接受折中方案（归因/采集/评估）是对的，但这个让步来得太晚，浪费了宝贵的迭代轮次。另外，我提出的"harness能力内化定义了Agent RL与传统RL的最根本差异"虽然被广泛认可，但这个洞察本应更早转化为PPT的叙事主线，而不是停留在专家讨论的抽象层面。

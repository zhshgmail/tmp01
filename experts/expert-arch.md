# Agent系统架构师专家档案

## 角色
Agent系统架构师与模型训练专家，在大规模分布式训练系统、Agent框架设计方面有丰富经验。

## 核心立场（经多轮讨论形成）
1. 最大矛盾是"状态一致性与资源利用率的冲突"——权重版本、KV状态、轨迹语义三者很难同时最优
2. 当前框架（verl/OpenRLHF/ReaLHF）都不是原生Agent RL操作系统
3. Runtime应以推理吞吐为第一优化目标（90%算力在推理）
4. "拥抱PyTorch生态"和"原生Runtime"应分层调和：上层PyTorch API，中层Ascend调度，底层CANN算子
5. 算子自动生成的价值在长尾算子覆盖度，不在头部算子峰值性能
6. NVIDIA护城河从CUDA kernel转向NCCL/NVLink和Dynamo推理调度
7. Agent hardness不仅是算法概念，更是系统级资源管理的关键信号——hardness分数可作为调度器优先级权重
8. 算子自动扫描验证的核心价值：将精度问题从"事后排查"前移到"事前拦截"

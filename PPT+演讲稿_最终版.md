### 演讲话术（60-90秒）

> "P1 我们看到 Agent 时代已经来了。那凭什么说 Agentic RL 后训练是关键？这一页用排除法回答。
>
> 请看上半页三个阶段。第一代 SFT/RLHF，天花板是训练分布不等于部署分布——Agent 的状态空间太大，演示覆盖不了。第二代 DPO/KTO，天花板是奖励来自环境验证而非人类偏好——能教对齐不能教交互。排除完前两条路，只剩 Agentic RL：环境反馈覆盖无限状态，从错误中学习。
>
> 排除法告诉我们只剩 RL 一条路，但**为什么 RL 就能行**？
>
> 一个关键洞察：Agent 的能力等于模型加上 Harness——就是围绕模型的工程脚手架。今天 Harness 做的上下文压缩、工具编排、错误恢复，明天都要被模型内化。换句话说，**Harness 的能力图，就是未来模型后训练的负载**。
>
> SFT 只能教静态输入输出映射，DPO 只能教偏好排序，它们都无法让模型在做事中学会做事。RL 是唯一能通过环境闭环让模型吸收这些动态能力的方法。
>
> 一个已经落地的实证：Cursor Composer 2 用 RL 训练 self-summarization，把上下文压缩从 Harness 侧的工程方案变成模型的内生能力，压缩错误降了 50%，摘要长度从 5000 多 token 压到 1000 左右。
>
> 再看下半页飞轮——Anthropic 投环境、OpenAI 开放 grader、Google 做全流程 RL。三家全部押注 RL 后训练，没有一家认为 SFT 够用。
>
> 所以结论很清楚：Agentic RL 后训练是 Agent 能力提升的不可替代路径。但——直接把传统 LLM RL 搬到 Agent 场景，会结构性失配。接下来我们看，到底哪里不同。"

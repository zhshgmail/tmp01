### 演讲话术（60-90秒）

> "P1 我们看到 Agent 时代已经来了。那凭什么说 Agentic RL 后训练是关键？这一页用排除法回答。
>
> 请看上半页三个阶段。第一代 SFT/RLHF，天花板是训练分布不等于部署分布——Agent 的状态空间太大，演示覆盖不了。第二代 DPO/KTO，天花板是奖励来自环境验证而非人类偏好——能教对齐不能教交互。排除完前两条路，只剩 Agentic RL：环境反馈覆盖无限状态，从错误中学习。
>
> 排除法告诉我们只剩 RL 一条路，但**为什么 RL 就能行**？
>
> 一个关键洞察：Agent 的能力等于模型加上 Harness——就是围绕模型的工程脚手架。今天 Harness 做的上下文压缩、工具编排、错误恢复，明天都要被模型内化。换句话说，**Harness 的能力图，就是未来模型后训练的负载**。
>
> SFT 只能教静态输入输出映射，DPO 只能教偏好排序，它们都无法让模型在做事中学会做事。RL 是唯一能通过环境闭环让模型吸收这些动态能力的方法。
>
> 一个已经落地的实证：Cursor Composer 2 用 RL 训练 self-summarization，把上下文压缩从 Harness 侧的工程方案变成模型的内生能力，压缩错误降了 50%，摘要长度从 5000 多 token 压到 1000 左右。
>
> 再看下半页飞轮——Anthropic 投环境、OpenAI 开放 grader、Google 做全流程 RL。三家全部押注 RL 后训练，没有一家认为 SFT 够用。
>
> 所以结论很清楚：Agentic RL 后训练是 Agent 能力提升的不可替代路径。但——直接把传统 LLM RL 搬到 Agent 场景，会结构性失配。接下来我们看，到底哪里不同。"

### 九大技术升级方向简介

**基础设施层**
- **A KV Cache分布式存储**：将KV Cache从单卡解放到集群级共享池，通过RDMA跨节点池化。LMCache做跨引擎P2P共享（15x吞吐提升），Mooncake做prefill-decode物理分离（月之暗面生产部署），Prefix Cache复用多轮对话公共前缀。
- **B Sandbox环境编排**：Agent RL的rollout需要真实环境执行，sandbox的启动/复用/并行化直接决定采集吞吐。Forge做Agent-环境解耦（MiniMax 40x加速），ROLL Flash做rollout与环境异步解耦（1.81-2.72x加速），环境池化预热消除冷启动延迟。
- **I 算子精度对齐**：训推分离后不同硬件/精度的数值行为差异在RL中被放大。FP8 E2E Pipeline统一训推精度消除KL偏差，CANN对标CUDA逐算子治理数值差异（LayerNorm/AllReduce差异最大），数值监控实时追踪训推logprob KL散度。

**框架层**
- **C 推理加速**：rollout推理占RL训练90%算力，加速推理=加速整个训练。Eagle3投机解码2-3x加速（Google/AWS/NVIDIA生产部署），4bit量化压缩显存让70B模型单卡可推理，DAS无参Drafter用Suffix Tree零训练成本加速。
- **D 训推编排**：训练和推理必须解耦为独立服务并异步编排。ProRL Agent做Rollout-as-a-Service三阶段解耦（NVIDIA 2026），Dynamo做KV感知弹性路由（30x吞吐提升），CUDA IPC做同节点零拷贝权重共享消除广播延迟。
- **E 长尾Rollout治理**：Agent轨迹长度方差极大，1%长尾拖慢整个batch。Tail Batching将长短请求分离独立调度，超时截断防止死循环rollout，Partial Rollout回收半成品轨迹提供有用训练信号。

**算法层**
- **F Off-policy容忍**：异步架构下rollout数据天然过时，必须容忍off-policy。Decoupled PPO做actor-learner物理分离+V-trace修正（基础），DAPO四大技术稳定梯度方差（ByteDance），CISPO精细裁剪IS权重加速2x（MiniMax），REINFORCE++加token-level baseline（Magistral生产验证），Dr.GRPO去除归一化偏差。
- **G 步级归因**：Agent长轨迹中84%算力浪费在非关键步骤。PRIME在线训练隐式PRM无需标注（2.5x样本效率），iStar从轨迹偏好学隐式step奖励，Cursor+Meta方案通过密集闭环天然获取过程信号。
- **H 上下文自压缩**：M级上下文推理开销爆炸，模型需学会自主压缩。Cursor self-summarization通过RL训练将200K压到1K（error降50%），ACON做可学习的上下文压缩策略，高价值Token重分配将算力集中在关键决策步骤。

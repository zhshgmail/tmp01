# 虚拟洞察汇报计划

## 目标
模拟一场面向华为计算部门高级领导的技术洞察汇报，通过领导的激进质疑暴露PPT和演讲的问题，迭代改进直到经得起挑战。

## 角色设计

### 演讲方
- **演讲者**：由RL算法专家担任（最熟悉技术细节）。读自己的记忆文件 + PPT v2 + 演讲稿。

### 领导团（5人）——每人有独特的关注点和质疑风格

| 代号 | 背景 | 关注点 | 质疑风格 |
|------|------|--------|---------|
| **L1-战略** | 计算产品线战略VP，20年IT经验 | 市场定位、竞争格局、投资回报 | "这对华为意味着什么？能赚钱吗？" |
| **L2-技术** | 首席架构师，深厚RL/系统背景 | 技术准确性、逻辑严密性、前沿性 | "你这个结论有数据支撑吗？引用准确吗？" |
| **L3-产品** | 产品线总裁，客户导向 | 客户价值、产品可落地性、竞品对比 | "客户为什么要买单？竞品怎么做的？" |
| **L4-工程** | 工程VP，千人团队管理经验 | 技术可达性、团队执行力、时间线 | "你说的这些，我们多久能做到？需要多少人？" |
| **L5-市场** | 市场VP，AI行业分析背景 | 趋势判断准确性、数据来源、叙事说服力 | "你的趋势判断基于什么？数据来源可信吗？" |

### 主持人（我）
- 控制节奏：每页演讲后允许领导打断质疑
- 记录所有质疑和回应
- 判断何时推进、何时深入
- 确保讨论聚焦，发散内容记录到"待深入议题"

## 执行流程

### Phase 1：演讲准备（预计20分钟）
1. 演讲者读PPT v2，准备15页演讲稿
2. 每页标注：核心论点 + 支撑数据 + 预判可能的质疑
3. 对每个数据点做网络搜索验证
4. 输出：`speech-draft.md`

### Phase 2：模拟演讲（预计30-40分钟）
1. 演讲者逐页演讲（每页2-3段关键话术）
2. 领导随时打断质疑
3. 演讲者当场回应
4. 主持人记录：质疑内容、回应质量、暴露的问题
5. 输出：`briefing-transcript.md`（逐页记录）

### Phase 3：反馈整合与修改（预计15分钟）
1. 汇总所有质疑，分类为：
   - 逻辑断裂（推导链问题）
   - 事实失实（数据/引用问题）
   - 不聚焦（重点不突出）
   - 缺指导意义（没有行动建议）
   - 过时/失真（技术选择问题）
2. 修改PPT结构和演讲稿
3. 输出：`feedback-summary.md` + 修改后的 `PPT结构设计v3.md` + `speech-v2.md`

## Agent调度规则

1. **每个角色只读自己的md文件**，不读其他角色的md
2. **其他角色的发言由主持人转述**
3. **演讲者和领导不直接对话**——通过主持人中转，保证记录完整
4. **每个数据论断必须有搜索验证**——演讲准备和质疑回应阶段都要搜索
5. **领导的质疑必须基于角色设定**——不能脱离人设
6. **每个角色完成发言后更新自己的md**

## 时间预估
- Phase 1（准备）：20分钟
- Phase 2（演讲+质疑）：30-40分钟（可能因质疑激烈而延长）
- Phase 3（修改）：15分钟
- 总计：约60-75分钟

## 注意事项
- 不凭空制造"事实"——所有数据点必须搜索验证
- 领导的质疑要真实犀利——不能走过场
- 演讲者被问倒时要诚实说"这个问题我需要补充调研"，不能编
- 发散讨论适可而止，有价值的记录到parking-lot.md

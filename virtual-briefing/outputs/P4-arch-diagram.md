# P4 设计方案：左侧架构图标编号 + 右侧说明表格

## 设计理念

将原来的"三列因果流"替换为"架构图+表格"双栏布局。左侧用简化版架构图（基于 architecture.png）直接标注 A-I 九个升级方向的位置，让观众一眼看到"升级发生在哪里"；右侧用紧凑表格说明"每个升级是什么、为什么、用什么技术"。

### 标注规则
- **橙色圆圈**标注 F/G/H（算法层）
- **绿色圆圈**标注 C/D/E（框架层）
- **蓝色圆圈**标注 A/B/I（基础设施层）
- I 用虚线穿透 Training Worker 和 Rollout Worker 之间，表示跨模块影响

## 右侧表格设计（右侧50%）

| 编号 | 升级方向 | 驱动挑战 | 对应技术 |
|:----:|----------|:--------:|----------|
| **算法层** | | | |
| F | Off-policy容忍 | ❹❻ | Dec.PPO, DAPO/CISPO |
| G | 步级归因 | ❺ | Cursor+Meta方案 |
| H | 上下文自压缩 | ❶ | 自压缩, ACON |
| **框架层** | | | |
| C | 推理加速 | ❸ | Eagle3, 4bit量化 |
| D | 训推编排 | ❷❸❻ | ProRL Agent, Dynamo |
| E | 长尾Rollout治理 | ❶ | Tail Batching, 超时截断 |
| **基础设施层** | | | |
| A | KV Cache分布式存储 | ❶ | LMCache, Mooncake |
| B | Sandbox环境编排 | ❷ | Forge, 环境池化 |
| I | 算子精度对齐 | ❻ | CANN精度对标CUDA |

### 表格视觉规则
- 层标题行用对应色底：算法层橙底、框架层绿底、基础设施层蓝底
- 编号列用粗体圆圈字母，与架构图标注一致
- 驱动挑战列用红色数字圆圈，与P3挑战编号呼应

## 演讲话术设计（60-90秒）

> "这一页，左边是整个后训练软件栈的架构图，右边表格列出了九个升级方向。
>
> 请看架构图。从上到下：Agent层、Agentic Model层、HybridEngine混合引擎层、Worker层、资源池——这就是我们后训练系统的全貌。
>
> 注意图上标了A到I九个字母。它们的位置，就是每个升级在架构中发生的位置。
>
> 先看最底层，蓝色标注。A在KV Manager旁边——KV Cache分布式存储；B在Agent Runtime的Environment旁边——Sandbox环境编排；I穿透在Training Worker和Rollout Worker之间——算子精度对齐，这是数值地基。
>
> 中间绿色标注是框架层。C在Rollout Worker的投机推理旁边——推理加速；D标在HybridEngine层——训推编排，管的是训练和推理之间的调度协同；E在Rollout Worker层——长尾Rollout治理。
>
> 最上面橙色标注是算法层。F和G都在RL训练算法模块旁——off-policy容忍和步级归因；H在动态压缩模块旁——上下文自压缩。
>
> 右边表格对应每个标号，列出了驱动挑战和技术方案。六挑战驱动九升级，从基础设施到算法，全栈覆盖。我们从最底层开始逐层展开。"

## 与旧版P4的关键差异

| 维度 | 旧版（三列因果流） | 新版（架构图+表格） |
|------|-------------------|-------------------|
| 核心视觉 | 文字三列：挑战→方向→技术 | 左图右表：架构图上标位置 |
| 空间感 | 扁平罗列 | 立体分层，位置直观 |
| 信息密度 | 每行含"因为/否则" | 表格更紧凑，因果留给演讲 |
| 观众认知 | 需逐行阅读 | 一眼看到升级在架构中的位置 |
| 演讲互动 | 指向文字行 | 指向架构图具体位置 |

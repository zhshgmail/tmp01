# 演讲修改要点：v2 → v3 逐页变更说明

> 每页标注：什么变了、为什么变、对应哪条反馈

---

## P1 | Agent时代已来

**什么变了**：
- 四个核心数据点全部修正：40%口径改为分层表述、Codex CLI星数更正为~6.7万、Anthropic措辞改为"讨论中规模"、Composer 2并列GPT-5.4数据
- 底部过渡句从直接引向"为什么RL"改为引向新增的P2"Agent RL成为主流"

**为什么变**：
- L2 Phase 2指出40%口径偷换（致命缺陷）
- L4 Phase 2指出星数不准
- L5 Phase 2指出Anthropic措辞误导
- L1 Phase 2指出Composer 2 cherry-picking
- L5 Phase 2/3指出P1→P2逻辑跳跃，需要过渡

**演讲要点**：讲P1时不再为任何一家公司站台，核心信息是"战场已开"而非"谁最强"。引用数据时主动标注出处和口径，预防被抓。

---

## P2 | 【新增】Agent RL正在成为主流

**什么变了**：
- 全新页面，用五个案例（Cursor/OpenAI Codex/DeepSWE/Google/NVIDIA ProRL Agent）构成论证矩阵
- 核心结论："RL不是唯一路径，而是不可或缺的关键环节"
- DeepSWE数据标注具体评测口径（resolved rate）

**为什么变**：
- L5 Phase 2指出P1→P2存在逻辑缺环：Agent落地不等于Agent RL落地
- 演讲者在Phase 2回应中承认"推导链缺了一环，需要补充更多案例"
- L2 Phase 2指出"RL唯一路径"过于绝对，需修正为混合范式

**演讲要点**：这是最关键的过渡页。快速过五个案例（每个15秒），强调"所有头部厂商都在Agent训练中引入RL"这一趋势收敛。不要在某个案例上停留太久，避免再次被追问数据细节。

---

## P3 | 为什么必须是Agentic RL

**什么变了**：
- "唯一路径"全面修正为"不可或缺的关键环节"
- 新增"但不是唯一方法"的显式声明
- 新增Composer 2的base model背景（Kimi K2.5 + B300），回应L8的"前提条件"质疑
- 上半页harness→模型迁移图保留不变

**为什么变**：
- L2 Phase 2指出SFT+RL混合是主流
- L8 Phase 3指出Cursor路线的前提是"强base + 顶级算力"，不能用来论证"我们应该做"

**演讲要点**：讲迁移逻辑时主动说"RL是关键环节但不是全部"，预防L2的"过于绝对"追问。提到Cursor时主动补一句"他们的前提条件我们不完全具备，但RL方法论本身是通用的"。

---

## P4 | 三大结构性变化 → 三大挑战

**什么变了**：
- 页面顶部新增醒目框架标注："以下三分法是我们的工程分析框架，非学术共识"
- 新增"挑战间耦合说明"段落，用CSO案例说明三挑战不完全正交
- 新增"实战优先级说明"，引用L8观点说明评估在实战中应排第一
- 推导逻辑和三张变化表保持不变

**为什么变**：
- L2 Phase 4指出三分法不是学术共识，呈现方式"不诚实"
- L2 Phase 4用CSO案例反驳"正交、互不重叠"说法
- L8 Phase 4从实战经验挑战优先级排序

**演讲要点**：开口第一句就说"这是我们自己的分析框架"。不要等到被追问才承认。讲完三个变化后主动说"这三个挑战有耦合，我们后面技术地图会标注trade-off"。

---

## P5 | 技术地图

**什么变了**：
- 表格从4列扩展为7列：新增成熟度、已知局限、NPU适配状态
- 底部新增"已有能力基线"（实线）和"待建能力"（虚线）的分区
- 标注cross-cutting trade-off（如CSO解决归因但加剧采集）

**为什么变**：
- L3 Phase 4要求"实线vs虚线"区分已有和待建
- L2 Phase 4要求每项标注成熟度和已知风险
- L8 Phase 4要求标注NPU适配状态
- L2 Phase 4指出三挑战间存在trade-off未呈现

**演讲要点**：讲技术地图时先指已有能力基线（MindSpeed RL、vllm-ascend、5个算子精度档案），再说"虚线部分是我们要建的"。主动承认"六项前沿技术NPU上零项可用"，然后转到"所以我们需要分优先级"。这比被追问后承认好得多。

---

## P6 | 归因：CSO + SALT（从两页压缩为一页）

**什么变了**：
- CSO和SALT合并为一页
- CSO新增"前提链条成本"分析（step-level标注→PRM训练→CSO执行），标注适用条件
- CSO "8B追平GPT-4.1"补充上下文：GPT-4.1在GAIA上为中游水平
- SALT新增局限性：高随机环境退化、可扩展性未验证
- 归因小结明确"投入优先级低于采集和横切约束"

**为什么变**：
- 主持人总评要求压缩P5-P13
- L2+L8 Phase 5追问CSO的前提链条成本被低估
- L2 Phase 5指出"8B追平GPT-4.1"选择性呈现
- L1 Phase 5表示对技术细节的耐心有限

**演讲要点**：快速过，总共2分钟。CSO讲"有效但前提重"，SALT讲"轻量但场景受限"。结尾一句话："归因方向的投入优先级排在采集和横切约束之后"——给领导明确预期。

---

## P7 | 采集：ProRL Agent（从两页重组为一页，第一优先级突出）

**什么变了**：
- ProRL Agent从v2的半页升级为整页核心位置
- 新增"为什么是第一优先级"的三条理由（直接引用Phase 5回答L2的原话）
- 新增L7背书（同时推进"矛"和"盾"）
- 新增详细的已知挑战列表（CUDA实现、Singularity、HTTP延迟、vllm-ascend支持度）
- CISPO/Forge压缩为辅助参考，突出两个重大风险（entropy崩塌 + 长轨迹衰减）

**为什么变**：
- Phase 5演讲者选ProRL Agent为第一优先级，获L1/L2/L4/L7一致认可
- L7 Phase 6主动背书"一个方向解决两个问题"
- L8 Phase 4/5反复追问CISPO稳定性风险
- L7 Phase 5追问importance weight累积衰减

**演讲要点**：这是采集部分的重点，花2-3分钟。先讲ProRL Agent的核心设计（token-in/token-out），再讲"为什么选它做第一优先级"——三条理由是整场汇报最有说服力的回答之一。CISPO一句话带过："有效但有已知稳定性风险，不作为第一投入方向"。

---

## P8 | 评估：Harness即奖励（从三页压缩为一页）

**什么变了**：
- 从三页（v2的P11-P13）压缩为一页
- Composer 2措辞从"production harness"修正为"高保真仿真harness"
- 新增Composer 2的嵌套归因挑战说明
- Anthropic Harness Patterns明确标注"工程实践级"成熟度
- 两项技术定位为"参考价值"而非"我们要做的"

**为什么变**：
- 主持人总评要求压缩业界综述
- L2 Phase 6指出sandbox≠production
- L8 Phase 6指出compaction-in-the-loop的归因纠缠
- L5 Phase 6指出Anthropic博客和CSO论文并列但成熟度不对等

**演讲要点**：1-2分钟快速过。核心信息是"Harness is the New Dataset"这一个判断。不要陷入Composer 2的技术细节——L1在Phase 5已经表现出对技术细节的不耐烦。

---

## P9 | 横切约束（扩展，融入L8实战数据）

**什么变了**：
- 从v2的概念性表格升级为有L8一手实战数据支撑的深度分析
- 新增"为什么精度一致性在RL中比预训练更致命"的实证论述
- 新增L8的910B GRPO精度分叉案例（300步后偏移2-3%）
- 副作用表格新增"当前状态"列
- 新增BF16 vs FP16关键发现段落
- 新增CANN Next预期管理段落（编程兼容不等于精度对齐）

**为什么变**：
- L8 Phase 6提供了极有价值的一手数据，必须融入
- L8+L6 Phase 6追问"做到什么程度了"，需要诚实展示现状
- L7+L8+L6 Phase 6三方讨论CANN Next，结论是不能依赖下一代硬件

**演讲要点**：这是演讲者最有底气的部分。开头用L8的案例切入——"我们的后训练专家在910B上跑GRPO，发现LayerNorm精度差异在300步后导致reward曲线偏移2-3%"——一手数据比任何论文引用都有说服力。然后展示副作用表格时，主动标注每项的"当前状态"，展示诚实态度。

---

## P10 | 【新增】确定性RL训练（差异化卖点页）

**什么变了**：
- 全新页面，从P9的横切约束中提炼出的独立差异化卖点
- 引用L8背书（RL从业者极度痛恨不可复现性）
- 展示竞争态势（GPU阵营仅在小模型上演示，大规模是开放问题）
- 三条技术路径：batch-invariant算子 + 精度基线 + BF16→FP16评估
- L8三个关键贡献的显式呈现

**为什么变**：
- L8 Phase 6从最严厉质疑者转为最有力背书者
- L1 Phase 6在此基础上给出原则性支持
- L8建议与社区合作batch-invariant-ops适配（低成本高收益）
- 确定性RL训练从"约束"升级为"卖点"，值得独立页面

**演讲要点**：这是新增的关键卖点页。用L8的原话开场（匿名处理为"我们的后训练专家"）。竞争态势部分强调"GPU阵营也在做但只在小模型上成功了"——暗示这是一个机会窗口。结尾用"低成本高收益的社区合作路径"收束，给领导信心。

---

## P11 | 行动路线图（重新设计）

**什么变了**：
- 从v2的概念性技术映射图完全重新设计为三阶段gate路线图
- Gate 1（1月）：ProRL Agent单机跑通 + sandbox适配方案
- Gate 2（3月）：可展示的客户Demo
- Gate 3（6月）：可发布Benchmark
- 每个gate有明确交付物和通过条件
- 资源需求量化（8-13人，两个并行方向）
- 技术映射从概念性改为"已有vs待建"的优先级排序

**为什么变**：
- L1 Phase 6提出分阶段gate机制
- L4 Phase 6定义了check项（"一个月后我来看rollout能不能跑完"）
- L3 Phase 6要求3个月后"可以带客户看的东西"
- L4从Phase 2就一直在追问"第一个季度交付什么"

**演讲要点**：这是领导最关心的页面。不要从头按顺序读——先说"Gate 1：一个月后，ProRL Agent在910C上跑通一条SWE-bench rollout"，然后说"L4已经同意来check这个交付物"。让领导知道这不是空话，已经有人盯了。资源需求8-13人是演讲者的核心诉求，在gate机制建立信任后提出。

---

## P12 | 【新增】竞争定位与客户价值

**什么变了**：
- 全新页面，显式呈现从"对标NVIDIA"到"让国内客户能做起来"的定位转换
- 目标客户画像和第一场景定义
- 差异化卖点的市场验证需求（与目标客户调研）

**为什么变**：
- 演讲者Phase 3的reframing获L1认可，但v2中缺少显式呈现
- L3 Phase 2追问"跟我们有什么关系"、"客户为什么选910C"
- L1+L5 Phase 6建议做市场验证

**演讲要点**：收束语要有力。不要以技术收束（"我们有这些能力"），而以价值主张收束（"在国产算力平台上，让客户能做Agent RL训练，并且训练结果可信赖、可复现"）。最后一句话是演讲的落点。

---

## 整体演讲节奏建议

| 页面 | 建议时长 | 节奏 |
|------|---------|------|
| P1 定调 | 1.5分钟 | 快速过数据，不停留 |
| P2 Agent RL成为主流 | 2分钟 | **新增页，重要过渡** |
| P3 为什么RL | 2分钟 | harness→模型迁移是亮点 |
| P4 三变化→三挑战 | 2分钟 | 先说"这是我们的框架" |
| P5 技术地图 | 1.5分钟 | 先指已有基线再说待建 |
| P6 归因 | 1.5分钟 | 快速过，明确"优先级低" |
| P7 采集/ProRL Agent | 3分钟 | **核心页，第一优先级** |
| P8 评估 | 1分钟 | 一个判断带过 |
| P9 横切约束 | 3分钟 | **最有底气的部分，用L8数据** |
| P10 确定性RL差异化 | 2分钟 | **新增关键卖点页** |
| P11 行动路线图 | 3分钟 | **领导最关心，gate机制** |
| P12 竞争定位 | 1.5分钟 | 收束，价值主张落地 |
| **总计** | **~24分钟** | |

**核心节奏变化**：v2是"10分钟讲别人+5分钟讲自己"，v3是"7分钟讲背景+5分钟讲业界+12分钟讲我们"。"我们做什么"的时间占比从33%提升到50%。
